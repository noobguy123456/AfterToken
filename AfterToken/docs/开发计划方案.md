## 二、开发阶段划分

### 阶段一：战斗闭环与配置化（优先级：P0）

**目标**：让战斗主循环真正可玩，所有核心数值走配置表，胜负条件完整。

| 任务 | 说明 | 依赖 | 产出/验收 |
|------|------|------|----------|
| 1.1 验证 Luban JSON 生成 | 当前 `Configs/GameConfig/` 配置工程已搭建，生成脚本已切换为 `cs-newtonsoft-json` + `json`；运行 `gen_code_bin_to_project.bat` 生成代码与数据。 | `.NET SDK 8.0+`、完整 Luban 工具链、YooAsset 收集器 | 导表命令可一键运行；生成代码进入 `GameProto/GameConfig/`，数据进入 `AssetRaw/Configs/json/`。 |
| 1.2 补齐事件接口 | 新增 `ILevelEvent`（波次/胜负）、`IBattleResultEvent`（胜利/失败/奖励）、共享层事件接口。 | TEngine 事件系统 Source Generator | `BattleSystem` 能广播胜利/失败；`LevelSystem` 能广播波次事件。 |
| 1.3 完善 `LevelSystem` | 按 `TbLevel` + `TbWave` 控制波次生成、胜负判定、时间限制。 | 配置表系统、事件接口 | 进入战斗后按波次刷怪，清完波次或玩家死亡触发胜负。 |
| 1.4 完善 `EnemySystem` | 敌人 FSM（Idle/Chase/Attack/Dead）、AI 类型配置、简单寻路/追踪。 | 配置表系统、对象池 | 敌人能追踪玩家、攻击、死亡；不同 configId 行为可配置。 |
| 1.5 完善 `BattleSystem` | 接入暴击、击杀事件、死亡统计。 | 共享系统事件接口 | `IBattleEvent.OnEntityKilled` 正确触发；`IBattleResultEvent` 能驱动结算。 |
| 1.6 完善 `PoolSystem` | 按类型拆分池（Projectile/Enemy/Effect/DamageText），支持 Preload/ClearAll。 | 对象池基础设施 | 高频对象复用，减少 Instantiate。 |
| 1.7 Play Mode 全流程验证 | 从 `main.unity` 启动，验证 MainMenu → Lobby → Battle → Result。 | 上述全部 | 无明显报错，流程可循环。 |

**阶段一里程碑**：战斗玩法可跑通，数值由配置表驱动。

## 四、技术约束与红线

1. **异步优先**：IO/资源加载用 `UniTask`，禁止 `Coroutine`、禁止同步加载大资源。
2. **模块访问**：通过 `GameModule.XXX` 访问框架，禁止 `ModuleSystem.GetModule<T>()` 重复查找。
3. **资源释放**：`LoadAssetAsync<T>` 必须配对 `UnloadAsset`；GameObject 用 `LoadGameObjectAsync`。
4. **热更边界**：`GameScripts/Main` 不热更，`GameScripts/HotFix/` 全部热更；禁止热更代码引用主包 internal 类型。
5. **事件解耦**：系统间用 `GameEvent` 接口事件，UI 内部用 `AddUIEvent`。
6. **禁止模式**：`Resources.Load`、`Instantiate(prefab)`、`FindObjectOfType`、手写事件 ID 硬编码。
7. **配置驱动**：新数值/新内容优先走 Luban 配置表，不在代码中硬编码。

## 五、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| Luban JSON 生成后 Tables 构造函数签名不匹配 | 中 | 根据实际生成的 `Tables.cs` 调整 `ConfigSystem.LoadJson` 签名。 |
| Luban 接入阻塞大量模块 | 高 | 第一周优先完成；如复杂度过高，先保持硬编码接口，再逐步替换。 |
| HybridCLR AOT 泛型问题 | 高 | 每阶段结束后打包验证，及时补充 AOT Generic References。 |
| Play Mode 全流程未在线验证 | 中 | 阶段一必须通过 Play Mode 验证，不推进到阶段二。 |
| 经营模块开发量较大 | 中 | 阶段四可进一步拆分，先实现最小可玩经营循环（建筑+生产+订单）。 |
| 事件接口 Source Generator 异常 | 中 | 任何新增 `[EventInterface]` 后必须强制刷新编译并检查 Console。 |

# 俯视角 2D 射击游戏 + 模拟经营功能设计

> 基于 TEngine（HybridCLR + YooAsset + UniTask + Luban）框架的功能环节与解决方案梳理。核心玩法为俯视角 2D 射击，扩展玩法为模拟经营，两者通过共享玩家数据和事件总线连接。

---

## 一、框架基础设施层

### 1. 项目结构与程序集划分

| 层级 | 目录/程序集 | 用途 | 射击游戏内容 |
|------|------------|------|-------------|
| 主包不可热更 | `Assets/GameScripts/`（Assembly-CSharp） | 启动入口、主包流程 | `GameEntry.cs`、`Procedure/` 保持不变 |
| 热更业务 | `Assets/GameScripts/HotFix/GameLogic/` | 游戏核心逻辑 | 射击、模拟经营、玩家档案、共享数据等 |
| 热更配置 | `Assets/GameScripts/HotFix/GameProto/` | Luban 生成代码 | 武器表、敌人表、关卡表、buff 表、建筑表、生产表、经营资源表等 |
| 框架核心 | `Assets/TEngine/Runtime/` | 模块系统 | Timer/Resource/UI/Scene/Audio/Fsm 等 |

**方案**：所有 gameplay 代码写在 `GameLogic` 热更程序集，通过 `GameModule.xxx` 访问框架能力。

---

### 2. 热更与资源包流程

- 主包流程已完成：`ProcedureLaunch → ProcedureSplash → ProcedureInitResources → ProcedureInitPackage → ProcedureCreateDownloader → ProcedureDownloadFile → ProcedurePreload → ProcedureLoadAssembly → ProcedureStartGame`
- 热更入口 `GameApp.Entrance(object[])` 中先调 `GameEventHelper.Init()`
- 资源全部放在 `Assets/AssetRaw/` 下，YooAsset 按 location 加载

**射击游戏涉及**：

- 角色 Prefab：`AssetRaw/Actor/Player.prefab`、`Enemy_XXX.prefab`
- 武器/子弹 Sprite 和 Prefab：`AssetRaw/Weapons/`、`AssetRaw/Projectiles/`
- 场景：`AssetRaw/Scenes/BattleScene.unity`
- UI Prefab：`AssetRaw/UI/Prefabs/BattleMainUI.prefab`
- 音效：`AssetRaw/Audios/BGM/`、`AssetRaw/Audios/SFX/`
- 配置 bytes：`AssetRaw/Configs/bytes/`
- 经营场景：`AssetRaw/Scenes/SimulationScene.unity`
- 经营建筑/作物 Prefab：`AssetRaw/Simulation/Buildings/`、`AssetRaw/Simulation/Crops/`
- 经营 UI：`AssetRaw/UI/Prefabs/SimulationMainUI.prefab`

## 二、核心 Gameplay 功能

### 3. 输入系统

**功能**：WASD 移动、鼠标瞄准、左键射击、滚轮/数字键切枪、R 换弹、Shift 冲刺。

**方案**：

- 使用 **Unity Input System**（推荐）或 Legacy Input
- 封装 `InputSystem` 单例（热更域），每帧读取输入并转换为 `GameEvent` 事件
- 输入事件定义接口事件：

```csharp
[EventInterface(EEventGroup.GroupBattle)]
public interface IBattleInputEvent
{
    void OnMoveInput(Vector2 dir);
    void OnAimInput(Vector2 worldPos);
    void OnFirePressed();
    void OnReloadPressed();
    void OnWeaponSwitch(int slot);
}
```

**注意**：输入层只发事件，不直接控制玩家，解耦业务逻辑。

---

### 4. 玩家角色（Player）

**功能**：移动、旋转朝向、换武器、受击、死亡、复活。

**方案**：

- 创建 `PlayerEntity` MonoBehaviour，挂载在 Player Prefab 上
- 使用 `Rigidbody2D` + `Collider2D` 做 2D 物理移动
- 玩家状态用 **FsmModule** 状态机：

```csharp
GameModule.Fsm.CreateFsm<PlayerEntity>("PlayerFsm", player,
    new PlayerIdleState(), new PlayerMoveState(), new PlayerDodgeState(), new PlayerDeadState());
```

- 玩家属性（HP、移速、防御）用 Luban 配置 + 运行时 `PlayerAttributeComponent`

---

### 5. 摄像机系统

**功能**：摄像机跟随玩家、平滑阻尼、边界限制、冲刺/受击时轻微抖动。

**方案**：

- 场景中 Main Camera 用 `CinemachineVirtualCamera`（若已安装 Cinemachine）
- 或在 `CameraSystem` 中每帧 `Vector2.SmoothDamp` 跟随玩家位置

```csharp
Vector3 targetPos = _player.position + Vector3.back * 10;
transform.position = Vector3.SmoothDamp(transform.position, targetPos, ref _velocity, _smoothTime);
```

---

### 6. 武器系统

**功能**：多武器槽、射速、弹夹、换弹、散布、后坐力、伤害、子弹类型。

**方案**：

- Luban 配置 `TbWeapon`：`id`, `name`, `fireRate`, `clipSize`, `reloadTime`, `damage`, `bulletId`, `spread`, `projectileType` 等
- 武器组件 `WeaponComponent` + `WeaponInstance`
- 武器类型枚举：射线（Hitscan）、实体弹（Projectile）、霰弹（MultiProjectile）、激光（Beam）
- 射击逻辑在 `WeaponInstance.Fire()` 中，使用 `GameModule.Timer` 控制射速 CD

```csharp
public abstract class WeaponBase
{
    public WeaponConfig Config;
    public float LastFireTime;
    public int CurrentAmmo;
    public abstract void Fire(Vector2 origin, Vector2 direction);
}
```

---

### 7. 子弹/投射物系统

**功能**：子弹飞行、穿透、反弹、追踪、爆炸、生命周期。

**方案**：

- `ProjectileEntity` MonoBehaviour + `ProjectileData`
- 使用 **ObjectPoolModule** 或对象池避免频繁 Instantiate/Destroy
- 飞行更新用 `UpdateDriver` 或 `GameModule.Timer`
- 碰撞检测用 `Physics2D.OverlapCircle` / `Raycast2D`

**资源加载**：

```csharp
var bulletGo = await GameModule.Resource.LoadGameObjectAsync("Bullet_Normal", parent: _projectileRoot);
Destroy(bulletGo); // 框架自动归还引用计数
```

---

### 8. 敌人与 AI

**功能**：巡逻、追踪玩家、攻击、闪避、死亡、掉落。

**方案**：

- `EnemyEntity` + `EnemyAttributeComponent`
- AI 用 **FsmModule** 状态机：
  - `EnemyIdleState`
  - `EnemyChaseState`
  - `EnemyAttackState`
  - `EnemyDeadState`
- 感知系统 `EnemyPerception`：视野范围、攻击范围
- 寻路：简单朝向玩家移动，或接入 **A* Pathfinding Project** / Unity NavMeshPlus（2D）
- 敌人生成用 `EnemySpawnerSystem`，按波次/时间生成

---

### 9. 战斗系统

**功能**：伤害计算、命中判定、暴击、Buff/Debuff、死亡事件、击杀统计。

**方案**：

- `BattleSystem` 单例处理伤害计算
- 伤害信息用 `MemoryPool`：

```csharp
public class DamageInfo : IMemory
{
    public int AttackerId;
    public int TargetId;
    public float Damage;
    public bool IsCritical;
    public void Clear() { /* 归还时重置 */ }
}
```

- 命中后广播事件：

```csharp
GameEvent.Get<IBattleEvent>().OnEntityDamaged(damageInfo);
GameEvent.Get<IBattleEvent>().OnEntityKilled(attackerId, targetId);
```

---

### 10. 物理与碰撞

**功能**：玩家/敌人/子弹/障碍物碰撞，触发区域（毒圈、陷阱）。

**方案**：

- Unity 2D Physics：`Rigidbody2D`（Dynamic/Kinematic/Static）+ `Collider2D`
- Layer 配置：`Player`、`Enemy`、`Projectile`、`Obstacle`、`Pickup`
- 使用 `Physics2D` 进行射线检测：

```csharp
RaycastHit2D hit = Physics2D.Raycast(origin, dir, maxDist, _enemyLayer);
```

- 碰撞事件在 `OnTriggerEnter2D` / `OnCollisionEnter2D` 中处理，转换为 `GameEvent` 通知战斗系统

---

## 三、模拟经营系统

### 11. 经营基础系统

**功能**：建筑建造与升级、资源产出、工人分配、经营时间管理。

**方案**：

- 独立模块 `Simulation/`，不依赖射击战斗的具体逻辑
- `BuildingSystem`：管理建筑的生命周期（建造中、生产中、闲置）
- `ProductionSystem`：处理资源生产队列和产出结算
- `WorkerSystem`：管理工人/居民，分配到不同建筑
- `SimTimeSystem`：经营场景的时间推进（可加速、暂停）
- 所有建筑和生产配置用 Luban `TbBuilding`、`TbProduction`

```csharp
public class BuildingSystem
{
    public void Build(int buildingId, Vector2Int gridPos);
    public void Upgrade(int buildingId);
    public void AssignWorker(int buildingId, int workerId);
}
```

---

### 12. 经营资源与经济

**功能**：金币、材料、作物、产品、交易订单。

**方案**：

- `CurrencySystem` / `InventorySystem` 作为共享模块，被射击和经营共同使用
- 经营产出进入玩家背包/仓库
- 经营消耗金币、材料、体力
- 订单系统 `OrderSystem`：接受订单，交付产品，获得奖励

```csharp
public class InventorySystem
{
    public bool AddItem(int itemId, int count);
    public bool ConsumeItem(int itemId, int count);
    public int GetItemCount(int itemId);
}
```

---

### 13. 经营与射击的联动

**功能**：战斗掉落经营资源，经营产出强化战斗。

**方案**：

- 战斗胜利奖励包含经营资源（材料、金币）
- 经营建筑可生产武器强化材料、弹药、消耗品
- 角色训练建筑可提升玩家战斗属性
- 通过共享事件通信，不直接调用对方系统：

```csharp
// 战斗胜利后，奖励系统分发资源
GameEvent.Get<IBattleResultEvent>().OnBattleWin(levelId, reward);

// 经营系统中升级武器
GameEvent.Get<IWeaponEvent>().OnWeaponUpgraded(weaponId);
```

## 四、关卡与游戏循环

### 11. 关卡/场景系统

**功能**：战斗场景加载、出生点、波次配置、胜利失败条件。

**方案**：

- 场景通过 `GameModule.Scene.LoadSceneAsync("BattleScene")`
- Luban 配置 `TbLevel`、`TbWave`
- `LevelSystem` 控制波次生成：

```csharp
GameModule.Timer.AddTimer(SpawnNextWave, time: wave.DelayTime);
```

- 胜利/失败切换流程状态：

```csharp
GameModule.Procedure.ChangeState<ProcedureBattleResult>();
```

---

### 12. 掉落与拾取系统

**功能**：敌人死亡掉落金币/经验/武器/道具，玩家拾取。

**方案**：

- `PickupEntity`：金币、血包、弹药箱、经验球
- 拾取用 Trigger 检测，拾取后发送 `GameEvent.Get<IBattleEvent>().OnPickupCollected(pickupId)`
- 掉落表用 Luban `TbDrop`

---

### 13. 经验/升级系统（可选）

**功能**：击杀获得经验、升级选择 buff/技能。

**方案**：

- `PlayerLevelComponent`
- 升级选项用 Luban `TbLevelReward`
- 选择界面用 `LevelUpUI`（UIWindow）

---

## 五、UI 系统

### 15. 战斗主界面

**功能**：血条、弹药、武器图标、小地图、波次/倒计时、击杀数、经验条。

**方案**：

- `BattleMainUI : UIWindow`，标记 `[Window(UILayer.UI, "BattleMainUI")]`
- 子组件：
  - `HpBarWidget`
  - `AmmoWidget`
  - `WeaponSlotWidget`
  - `MinimapWidget`
  - `WaveInfoWidget`
- 通过 `AddUIEvent` 监听事件刷新 UI：

```csharp
protected override void RegisterEvent()
{
    AddUIEvent<int, int>(IBattleEvent_Event.OnHpChanged, OnHpChanged);
    AddUIEvent<int, int>(IBattleEvent_Event.OnAmmoChanged, OnAmmoChanged);
}
```

### 16. 经营主界面

**功能**：建筑列表、资源显示、生产队列、工人分配、订单板、地图切换。

- 子组件：
  - `BuildingWidget`
  - `ResourceWidget`

## 六、资源与性能

### 16. 资源加载规范

| 资源类型 | API | 释放方式 |
|---------|-----|---------|
| Sprite/图集子图 | `image.SetSprite("icon")` | 无需手动 |
| 角色/特效/子弹 Prefab | `LoadGameObjectAsync` | Destroy 自动 |
| TextAsset/ScriptableObject | `LoadAssetAsync<T>` | `UnloadAsset` |
| 场景 | `GameModule.Scene.LoadSceneAsync` | `UnloadAsync` |

**注意**：射击游戏子弹频繁生成，禁止 `Instantiate(prefab)`，应使用对象池。

---

### 17. 对象池

**功能**：复用子弹、敌人、特效、伤害数字。

**方案**：

- 使用 `GameModule.Resource` 的 `LoadGameObjectAsync` 配合自定义对象池
- 或封装 `GameObjectPool<T>` 管理启用/禁用
- 高频 C# 对象（DamageInfo、ProjectileData）用 `MemoryPool`

---

### 18. 音频系统

**功能**：BGM、射击音效、受击音效、UI 音效、环境音。

**方案**：

```csharp
// BGM
GameModule.Audio.Play(AudioType.Music, "BGM_Battle", bLoop: true);

// SFX
GameModule.Audio.Play(AudioType.Sound, "SFX_Gunfire");
GameModule.Audio.Play(AudioType.UISound, "UI_Click");
```

---

### 19. 特效系统

**功能**：枪口火焰、子弹轨迹、命中火花、爆炸、死亡溶解。

**方案**：

- 特效 Prefab 放在 `AssetRaw/Effects/`
- 使用对象池或 `LoadGameObjectAsync` + `Destroy`（短期特效）
- 可用 Unity Particle System，或后续接入 VFX

## 七、数据与配置

### 20. Luban 配置表

需要配置的表（在 `TEngine/Configs/GameConfig/Datas/`）：

| 表名 | 内容 |
|------|------|
| `TbWeapon` | 武器基础属性 |
| `TbBullet` | 子弹飞行速度、穿透、反弹、生命周期 |
| `TbEnemy` | 敌人属性、AI 类型、掉落 |
| `TbWave` | 波次敌人组成、生成间隔 |
| `TbLevel` | 关卡配置、胜利条件 |
| `TbDrop` | 掉落概率 |
| `TbBuff` | Buff/Debuff 效果 |
| `TbPlayerAttr` | 玩家初始属性、成长 |
| `TbBuilding` | 建筑配置：建造成本、产出、时间、工人上限 |
| `TbProduction` | 生产配方：消耗材料、产出物品、生产时间 |
| `TbCrop` | 作物配置：生长时间、季节、产出 |
| `TbOrder` | 订单配置：需求物品、奖励、刷新条件 |
| `TbItem` | 通用物品配置：类型、堆叠上限、用途 |
| `TbPlayerAnimation` | 玩家状态与动画映射 |

**访问**：

```csharp
var weaponCfg = ConfigSystem.Instance.Tables.TbWeapon.Get(weaponId);
```

## 八、扩展与优化

### 22. 调试与 GM

**功能**：无敌、秒杀、跳关、刷怪、显示碰撞盒。

**方案**：

- 使用 TEngine 内置 `DebuggerModule`（`~` 键呼出）
- 扩展自定义 Debugger 窗口
- 或开发 `GMCommandSystem` 解析聊天框指令

---

### 23. 性能优化

- 2D 渲染：Sprite Atlas 合批、控制 Overdraw
- 对象池减少 GC
- 分帧生成敌人，避免同帧大量 Instantiate
- 使用 `GameModule.Timer` 替代大量 `Update`
- 限制同屏子弹/敌人数量
- 场景切换后调用 `GameModule.Resource.UnloadUnusedAssets()`

---

### 24. 多平台与热更

- 资源包按平台构建：`StandaloneWindows64` 已有
- 热更时下载 weapon/bullet/enemy 配置和 DLL，无需重新出包即可调整数值和逻辑
- 注意 AOT 泛型补全：常见如 `List<DamageInfo>`、`UniTask<GameObject>` 等

---

## 推荐开发顺序

1. 搭建共享层：`PlayerProfileSystem`、`InventorySystem`、`CurrencySystem`
2. 搭建基础场景 + Player 移动 + 摄像机跟随
3. 实现武器射击 + 子弹对象池
4. 实现敌人 AI + 生成器
5. 接入 Luban 配置表（武器、敌人、关卡、物品）
6. 战斗系统（伤害、死亡、击杀）
7. 战斗 UI 主界面 + 事件绑定
8. 音效 + 特效
9. 关卡流程 + 胜利失败
10. 模拟经营基础系统（建筑、生产、资源）
11. 经营 UI 主界面
12. 战斗与经营联动（奖励、武器强化、角色训练）
13. 存档 + 设置
14. 热更测试 + 性能优化

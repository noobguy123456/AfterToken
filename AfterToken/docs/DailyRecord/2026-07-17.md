1. **修复启动流程 Console 报错（两类）**
   - 报错 1：`Main logic type 'GameApp' missing.` 每帧刷屏。根因：未提交的命名空间重构（48 个文件）将 `GameApp` 包入 `namespace GameLogic`，而 `ProcedureLoadAssembly.cs:135` 仍按简单名 `GetType("GameApp")` 反射查找，返回 null 后 `Log.Fatal`；且 `OnUpdate` 无一次性守卫，失败路径每帧重试。
   - 修复：`ProcedureLoadAssembly.cs` 查找改为全限定名 `GameLogic.GameApp`。
   - 报错 2：`ArgumentException: An item with the same key has already been added. Key: GameLogic.IBattleEvent`（修复 1 后暴露）。根因：`AllAssemblyLoadComplete` 每帧调用 `GameApp.Entrance`；工作区新版 `EntranceAsync` 在 `await ConfigSystem.LoadAsync()` 处挂起数帧，期间 `GameEventHelper.Init()`（SourceGenerator 生成，位于 `Assets/TEngine/Runtime/Core/GameEvent/SourceGenerator.dll`）被重复执行，`EventMgr.RegWrapInterface<T>` 重复 Add 抛异常。
   - 修复：`ProcedureLoadAssembly` 新增 `_assemblyLoadFired` 一次性守卫，保证 `AllAssemblyLoadComplete`（即 `Entrance` 反射调用）只触发一次。
   - 涉及文件：`Assets/GameScripts/Procedure/ProcedureLoadAssembly.cs`
   - 验证：`AssetDatabase.Refresh` 重编译通过，Console 0 error / 0 warning。

2. **同类错误全局排查（结论：无其他同类问题）**

3. **补建 PlayerDeathUI.prefab（修复死亡弹窗断裂链路）**
   - 背景：`PlayerDeathUI.cs` 早已存在且声明 `location: "PlayerDeathUI"`，但 prefab 缺失，玩家死亡后弹窗加载失败，游戏停在 timeScale=0。

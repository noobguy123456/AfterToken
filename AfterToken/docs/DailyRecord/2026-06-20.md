# 2026-06-20

## 工作内容

1. **射击模块设计与实现**
   - 设计并实现俯视角 2D 射击核心系统：
     - `WeaponSystem`：武器槽管理、瞄准、开火调度。
     - `BallisticSystem`：Raycast / Projectile 弹道分发与子弹视觉。
     - `ProjectileSystem`：飞行物生成、飞行、命中判定与回收。
     - `AimAssistSystem`：辅助瞄准与火箭锁定。
     - `WeaponInstance`：武器运行时实例。
     - `ProjectileEntity`：飞行物实体。
   - 新增武器配置层：
     - `WeaponConfig` / `WeaponConfigMgr`（临时硬编码内存表）。
     - `Enums.cs` 中新增 `WeaponType`、`BallisticType`、`FireMode`、`AimMode` 等枚举。
   - 完善玩家战斗状态机：
     - `PlayerIdleState`、`PlayerMoveState`、`PlayerReloadState`、`PlayerDodgeState`、`PlayerDeadState`。
     - `PlayerStateBase` 统一输入处理与状态切换。
   - 新增射击相关 UI 与反馈：
     - `WeaponWheelUI` 武器轮盘。
     - `SniperScopeUI` 狙击镜 RenderTexture 效果。
     - `HitFeedbackSystem` 受击方向指示与命中反馈。
   - 定义射击/战斗事件接口：`IWeaponEvent`、`IProjectileEvent`、`IBattleInputEvent`、`IPlayerEvent`、`IHitFeedbackEvent`、`ICameraEvent`、`IEnemyEvent`、`IBattleEvent`。
   - 编写/更新 `docs/射击模块实现文档.md`，覆盖总体架构、目录结构、武器配置、开火流程、弹道与飞行物、辅助瞄准、命中反馈等章节。

2. **流程层重构**
   - 新增 `GameplayProcedureBase`，统一处理 LoadingUI 场景切换、取消令牌管理和 OnLeave 资源清理。
   - `ProcedureMainMenu`、`ProcedureLobby`、`ProcedureBattle` 改为继承 `GameplayProcedureBase`，通过 `LoadSceneWithLoadingAsync` 实现带 LoadingUI 的场景过渡。

3. **LoadingUI 集成**
   - `LoadingUI` 使用 `UILayer.System`，支持 `SetProgress(float)` 更新进度条与文本。
   - 所有游戏流程的场景加载均先显示 LoadingUI、异步加载场景、更新进度、完成后关闭 LoadingUI。

4. **主菜单 / 大厅 UI Prefab 化**
   - `MainMenuUI`、`LobbyUI` 改为 Prefab 驱动，`ScriptGenerator` 绑定节点引用，`OnCreate` 仅负责事件注册与动态数据填充。
   - 核对 Prefab 节点命名（`m_text_Title`、`m_rect_ButtonRoot/m_btn_Start`、`m_rect_LevelList/m_btn_LevelTemplate` 等）与代码绑定路径一致。

5. **编辑器工具与配置修正**
   - `BattleSceneSetup` 改为非破坏性更新已有 Prefab，新增 `CreateMainMenuScene()` 与 `CreateLobbyScene()`。
   - 修正 `LevelConfigMgr.GetAll()` 返回按 `id` 排序的关卡列表。
   - 新增 `Assets/Editor/ForceRecompile.cs`，提供 `Tools/Force Recompile` 菜单以解决热更 DLL 不自动刷新问题。

6. **文档补充**
   - 新增 `docs/CONTEXT.md`，记录项目架构、流程、UI 规范、已知技术债与最近变更。
   - 更新 `docs/战斗模块使用手册.md` 与 `docs/射击模块实现文档.md`，与当前实现保持一致。
   - 新增 ADR：`0001-loading-ui-scene-transition`、`0002-ui-prefab-driven-layout`、`0003-cross-procedure-data`、`0004-hotfix-procedure-takeover`。

7. **调试辅助**
   - 在流程进入、场景加载、UI 节点绑定等关键路径添加 `Log.Debug`，便于在 Play Mode 中定位可能的卡死位置。

8. **静态验证**

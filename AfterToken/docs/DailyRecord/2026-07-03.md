### 3. 自研 2D 网格 A* 寻路系统
为彻底解决敌人被障碍物阻挡后无法绕路的问题，设计并实现了一套低耦合、可插拔的寻路系统。

#### 3.1 架构设计
- **接口层**：`INavigationSystem` / `INavigationGridBuilder`
  - 业务代码只依赖接口，不依赖具体算法。
- **数据层**：`NavigationGrid`
  - 只读二维网格数据，包含 `Origin`、`CellSize`、`Width`、`Height`、`Walkable[]`。
- **算法层**：`AStarNavigationSystem`
  - 实现 A* 寻路，4 方向移动。
  - 使用自研 `SimplePriorityQueue<T>` 最小堆优先队列。
  - 支持起点/终点不可行走时的最近可行走点降级。
  - 路径平滑：通过 `Physics2D.Linecast` 去除共线路径点，只保留拐点。
- **生成层**：`ColliderGridBuilder`
  - 自动扫描 `Obstacle` 层所有 `Collider2D`。
  - 取障碍物 bounds 并扩展 `spawnRadius + margin` 作为网格边界。
  - 对每个格子做 `OverlapCircle` 判定是否被阻挡。
- **调度层**：`NavigationSystem` MonoBehaviour
  - 单例，负责管理导航网格、寻路算法、路径缓存。
  - 预留分帧调度接口（当前实现同步寻路 + 缓存）。

### 5. Review 后 P0/P1 整改
针对敌人 FSM 与寻路系统进行代码审查，完成以下解耦与性能优化：
- **解耦 `EnemyChaseState` 与 `NavigationSystem` 具体类**
  - `EnemyStateContext` 新增 `INavigationSystem NavigationSystem` 字段。
  - `EnemyStateMachineDriver.UpdateContext` 将导航系统接口注入黑板。
  - `EnemyChaseState.RefreshPath` 改为读取 `Context.NavigationSystem`，不再直接引用 `NavigationSystem` 单例。
  - `NavigationSystem` 实现 `INavigationSystem` 接口。
- **限制 `ColliderGridBuilder` 扫描范围**
  - 移除 `-1000~1000` 全图扫描。
  - 新增 `scanCenter` / `scanRadius` 参数，默认以玩家出生点为中心、生成半径 + 余量为边界。
  - `PlayerSystem` 新增 `SpawnPosition` 公共属性。
  - `ProcedureBattle` 初始化导航网格时传入玩家出生点。
- **敌人血条共享白色 Sprite**
  - `EnemyEntity` 使用静态 `_sharedWhiteSprite`，所有敌人复用，避免每实例创建 `Texture2D`。
- **A* 使用 generation array**
  - `AStarNavigationSystem` 移除每次寻路的 `Array.Fill`，改用 `_visited` + `_currentGeneration` 标记访问状态。
- **`PathResult.Failed` 共享只读实例**
  - 避免失败路径频繁时产生 GC。
- **`EnemyChaseState` fallback 移动统一时间参数**
  - `MoveTowards` 接受 `elapse` 参数，替代 `Time.deltaTime`。
- **清理 `NavigationSystem` dead code**
  - 移除未使用的分帧请求队列、`_maxRequestsPerFrame`、`_pathUpdateInterval`。
  - 缓存 key 量化从 `RoundToInt` 改为 `FloorToInt`，减少边界抖动。

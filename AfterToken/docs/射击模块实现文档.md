# 射击模块实现文档

> 适用范围：AfterToken 俯视角 2D 射击战斗（HotFix 层）。
> 基于 TEngine + YooAsset（EditorSimulateMode）+ UniTask + HybridCLR，2D 物理使用 `Rigidbody2D`/`Physics2D`。

---

## 1. 总体架构

```
GameApp.Entrance
    └── ProcedureMainMenu
            ├── ShowUIAsyncAwait<LoadingUI>()
            ├── LoadSceneAsync("MainMenuScene")   // 进度更新到 LoadingUI
            ├── CloseUI<LoadingUI>()
            └── ShowUIAsyncAwait<MainMenuUI>()
                    │ 开始游戏
                    ▼
            ProcedureLobby
                    ├── ShowUIAsyncAwait<LoadingUI>()
                    ├── LoadSceneAsync("LobbyScene")
                    ├── CloseUI<LoadingUI>()
                    └── ShowUIAsyncAwait<LobbyUI>()
                    │ 选择关卡
                    ▼
            ProcedureBattle
                    ├── ShowUIAsyncAwait<LoadingUI>()
                    ├── LoadSceneAsync(sceneName)   // 由 LevelConfig 决定
                    ├── InitializeBattleSystems()   // 创建 BattleRoot 并挂载所有 System
                    ├── CloseUI<LoadingUI>()
                    └── ShowUIAsyncAwait<BattleMainUI>()

BattleRoot
├── InputSystem          // 输入 → 事件
├── PlayerSystem         // 玩家创建 + 状态机 + 受伤
├── WeaponSystem         // 武器槽、瞄准、开火
├── AimAssistSystem      // 辅助瞄准 + 火箭锁定
├── BallisticSystem      // Raycast / Projectile 分发 + 子弹视觉
├── ProjectileSystem     // 飞行物生成、飞行、命中、回收
├── BattleSystem         // 伤害应用
├── EnemySpawnSystem     // 敌人生成（占位）
├── HitFeedbackSystem    // 受击方向指示 + 命中反馈
└── PoolSystem           // GameObject 对象池（通用）

Camera.main
└── CameraSystem         // 跟随、FOV、震动、狙击镜 RenderTexture
```

### 关键约定

- 所有跨系统通信均通过 **TEngine 事件接口**（`[EventInterface(EEventGroup.GroupLogic)]`）完成。
- 发布事件：`GameEvent.Get<IxxxEvent>().OnXxx(...)`。
- 订阅事件：使用 `GameEventMgr.AddEvent(IxxxEvent_Event.OnXxx, handler)`，并在 `OnDestroy` 中 `Clear()`。
- 资源通过 YooAsset `DefaultPackage` 加载，编辑器下依赖 `EditorSimulateModeHelper.SimulateBuild` 生成的模拟清单。
- 若资源加载失败，各系统会回退到运行时创建的占位体（Primitive Sprite），保证功能可跑通。

## 3. 事件接口（Event Interfaces）

所有接口标注 `[EventInterface(EEventGroup.GroupLogic)]`，因为当前 `EEventGroup` 只存在 `GroupUI` 和 `GroupLogic`，战斗逻辑事件统一走 `GroupLogic`。TEngine 会为每个接口生成伴生静态类 `IxxxEvent_Event`，里面包含事件 ID，用于 `GameEventMgr.AddEvent` 订阅。

### 3.1 输入事件 `IBattleInputEvent`

发布者：`InputSystem`。

| 方法 | 参数 | 说明 |
|------|------|------|
| `OnMoveInput` | `Vector2 direction` | WASD 归一化方向 |
| `OnAimInput` | `Vector2 worldPosition` | 鼠标世界坐标 |
| `OnFirePressed` | - | 左键按下 |
| `OnFireReleased` | - | 左键抬起 |
| `OnAimPressed` | - | 右键/瞄准键按下 |
| `OnAimReleased` | - | 右键/瞄准键抬起 |
| `OnReloadPressed` | - | 按 R |
| `OnWeaponSwitch` | `int delta` | 滚轮 ±1 |
| `OnWeaponWheelToggled` | `bool visible` | 武器轮盘打开/关闭 |
| `OnWeaponSelected` | `int slot` | 轮盘选中槽位 |
| `OnDodgePressed` | - | 按 Space |

### 3.2 武器事件 `IWeaponEvent`

发布者：`WeaponSystem`、`WeaponInstance`、`PlayerReloadState`。

| 方法 | 参数 | 说明 |
|------|------|------|
| `OnWeaponEquipped` | `ownerId, slot, weaponConfigId` | 装备武器到指定槽 |
| `OnStartFire` / `OnStopFire` | `ownerId` | 开始/停止持续开火 |
| `OnReload` | `ownerId` | 触发换弹 |
| `OnReloadStateChanged` | `ownerId, isReloading` | 换弹状态变化（开始/完成） |
| `OnWeaponSwitched` | `ownerId, slot` | 当前槽切换 |
| `OnAimStateChanged` | `ownerId, isAiming` | 瞄准状态变化 |
| `OnFire` | `origin, direction, weaponConfigId, ownerId` | 真正发射一发 |

### 3.3 玩家事件 `IPlayerEvent`

发布者：`PlayerEntity`、`PlayerSystem`、`PlayerDeadState`、`WeaponSystem`。

| 方法 | 参数 | 说明 |
|------|------|------|
| `OnPlayerCreated` | `Vector3 position` | 玩家创建完成 |
| `OnPlayerPositionChanged` | `Vector3 position` | 每 FixedUpdate 广播 |
| `OnPlayerStateChanged` | `stateName, prevStateName` | 状态机切换 |
| `OnHpChanged` | `currentHp, maxHp` | HP 变化 |
| `OnAmmoChanged` | `currentAmmo, maxAmmo` | 弹药变化 |
| `OnPlayerDamaged` | `damage, hitDirection` | 玩家受伤 |
| `OnPlayerDied` | - | 玩家死亡 |

### 3.4 战斗事件 `IBattleEvent`

发布者：`BallisticSystem`、`ProjectileSystem`。

| 方法 | 参数 | 说明 |
|------|------|------|
| `OnEntityDamaged` | `DamageInfo damageInfo` | 实体受到伤害 |
| `OnEntityKilled` | `int attackerId, int targetId` | 击杀（已定义，未触发） |
| `OnPickupCollected` | `int pickupId, int collectorId` | 拾取（预留） |

### 3.5 相机事件 `ICameraEvent`

发布者：`WeaponSystem`、`PlayerSystem`。

| 方法 | 参数 | 说明 |
|------|------|------|
| `OnCameraShake` | `magnitude, duration` | 相机震动 |
| `OnAimFovChanged` | `targetFov` | 瞄准 FOV 目标 |

### 3.6 敌人事件 `IEnemyEvent`

发布者：`EnemySpawnSystem`、`EnemyEntity`。

| 方法 | 参数 | 说明 |
|------|------|------|
| `OnEnemySpawned` | `int enemyId, int configId` | 敌人生成 |
| `OnEnemyDied` | `int enemyId` | 敌人死亡 |

### 3.7 反馈事件 `IHitFeedbackEvent`

发布者：`PlayerSystem`、`BallisticSystem`、`BattleSystem`、`AimAssistSystem`。

| 方法 | 参数 | 说明 |
|------|------|------|
| `OnDamageIndicator` | `float angle, float duration` | 8 方向受击指示 |
| `OnHitTarget` | `bool isCritical` | 命中标记 |
| `OnTargetLocked` | `int targetId` | 火箭锁定完成 |

### 3.8 子弹事件 `IProjectileEvent`

发布者：`ProjectileSystem`、`ProjectileEntity`。

| 方法 | 参数 | 说明 |
|------|------|------|
| `OnProjectileCreated` | `int projectileId, Vector3 position` | 子弹创建 |
| `OnProjectileHit` | `int projectileId, GameObject target` | 子弹碰撞 |
| `OnProjectileDestroyed` | `int projectileId` | 子弹销毁 |

## 5. 系统（Systems）

### 5.1 `InputSystem`

- 职责：把原始输入转换为战斗事件。
- 键位映射：

| 输入 | 事件 |
|------|------|
| WASD / 方向轴 | `OnMoveInput(Vector2)` |
| 鼠标位置 → 世界坐标 | `OnAimInput(Vector2)` |
| 左键按下/抬起 | `OnFirePressed` / `OnFireReleased` |
| 右键按下/抬起 | `OnAimPressed` / `OnAimReleased` |
| 滚轮向上/向下 | `OnWeaponSwitch(+1/-1)` |
| 按住 Tab | 打开 `WeaponWheelUI`，`Time.timeScale = 0.2`，`OnWeaponWheelToggled(true)` |
| 松开 Tab | 关闭轮盘，`OnWeaponSelected(slot)`，`OnWeaponWheelToggled(false)` |
| R | `OnReloadPressed` |
| Space | `OnDodgePressed` |

- 轮盘扇区计算（与 `WeaponWheelUI` 一致）：
  - 以屏幕中心为原点，计算鼠标角度 0-360°。
  - `<120°` → 槽 0，`120°-240°` → 槽 1，`≥240°` → 槽 2。

### 5.2 `WeaponSystem`

- 职责：武器槽管理、瞄准、开火、扩散、移速/灵敏度系数。
- 关键字段：
  - `MAX_WEAPON_SLOTS = 3`
  - `_slots[]`：`WeaponInstance[]`
  - `CurrentWeapon`：当前槽实例
  - `_isAiming` / `_isFiring` / `_firePending`
- 生命周期：
  - `Awake`：注册输入事件、设置单例。
  - `Start`：从 `PlayerSystem` 取 ownerId，装备 1001/1002/1003，切换到槽 0。
  - `Update`：Tick 当前武器（扩散恢复），处理 Auto 持续射击或 pending 单发。
- 开火流程：
  1. `OnFirePressed`：Auto 模式设 `_isFiring = true`；Single 模式设 `_firePending = true`。
  2. `Update` → `TryFire`。
  3. 取玩家位置与瞄准点，计算方向。
  4. 调用 `AimAssistSystem.ApplyAimAssist` 修正方向。
  5. 根据移动/瞄准状态计算 `spread`，随机偏移方向。
  6. 调用 `WeaponInstance.Fire`，触发 `IWeaponEvent.OnFire` 与 `OnAmmoChanged`。
  7. 触发相机震动 `OnCameraShake`。
- 瞄准：
  - `AimMode.Hold`：按住右键开镜，松开关闭。
  - `AimMode.Toggle`：每次按下切换。
  - 触发 `IWeaponEvent.OnAimStateChanged` 与 `ICameraEvent.OnAimFovChanged`。
  - 若当前武器是狙击枪，开镜时打开 `SniperScopeUI`，关闭时关闭。
- 切换武器：滚轮/轮盘/数字键，带 0.3s 冷却；切换时取消瞄准。
- 换弹：`WeaponSystem` 监听 `IWeaponEvent.OnReload`（由 `PlayerReloadState` 触发），调用 `CurrentWeapon.Reload(ownerId)`；不再直接响应 `OnReloadPressed`，避免在闪避/死亡/换弹期间非法换弹。换弹开始/完成时广播 `IWeaponEvent.OnReloadStateChanged(ownerId, isReloading)`。
- 公共 API：
  - `CurrentWeapon`、`IsAiming`、`IsFiring`
  - `GetCurrentMoveSpeedMultiplier()`：基础 * 开火系数（换弹/闪避时不应再乘，见 `PlayerSystem.Update`）
  - `GetCurrentAimSensitivityMultiplier()`
  - `IsAimAssistEnabled()`

### 5.3 `WeaponInstance`

- 单把武器的运行时数据。
- 字段：`Config`, `CurrentAmmo`, `CurrentSpreadIncrement`, `LastFireTime`, `IsReloading`, `IsFiring`。
- 方法：
  - `Tick(deltaTime, isMoving, isAiming)`：扩散自然恢复。
  - `CanFire(currentTime)`：未换弹、有弹药、超过射速间隔。
  - `Fire(origin, direction, ownerId)`：扣弹药、加扩散、触发 `IWeaponEvent.OnFire`、刷新弹药 UI；**弹匣打空后自动调用 `Reload(ownerId)`**。
  - `Reload(ownerId)`：设 `IsReloading = true`，广播 `OnReloadStateChanged(ownerId, true)`，通过 `GameModule.Timer` 在 `reloadTime` 后回满弹药，完成后广播 `OnReloadStateChanged(ownerId, false)`。
  - `CalculateSpread(isMoving, isAiming)`：`baseSpread + CurrentSpreadIncrement`，再乘移动/瞄准系数。

### 5.4 `PlayerSystem`

- 职责：加载玩家、玩家状态机、受伤/死亡。
- 生命周期：
  - `Awake`：订阅输入事件。
  - `Start`：异步 `CreatePlayerAsync()`。
  - `OnDestroy`：清理事件、销毁 FSM。
- 创建玩家：
  1. `GameModule.Resource.LoadGameObjectAsync("Player", _spawnPoint)`。
  2. 失败则创建占位玩家（青色 Sprite）。
  3. 确保 `PlayerEntity` 组件存在，调用 `ResetEntity()`。
  4. 创建 FSM：`PlayerIdleState/MoveState/DodgeState/ReloadState/DeadState`，启动 `PlayerIdleState`。
  5. 广播 `OnPlayerCreated` 与 `OnHpChanged`。
- 输入处理：
  - `OnMoveInput` → `PlayerEntity.SetMoveDirection`
  - `OnAimInput` → `PlayerEntity.SetAimPosition`
  - `OnReloadPressed` → FSM 数据 `NextState = "Reload"`（需 `CanSwitchFromCurrentState`）；进入 `PlayerReloadState` 后触发 `IWeaponEvent.OnReload`，由 `WeaponSystem` 执行实际换弹。
  - `OnDodgePressed` → FSM 数据 `NextState = "Dodge"`
- `Update`：
  - 若处于 `PlayerDodgeState`，不应用武器移速系数（避免覆盖闪避速度）。
  - 否则 `MoveSpeed = BaseMoveSpeed * WeaponSystem.GetCurrentMoveSpeedMultiplier()`。
- 受伤：
  - 扣 HP，广播 `OnHpChanged` / `OnPlayerDamaged`。
  - 计算相对玩家朝向的受击角度，触发 `OnDamageIndicator`。
  - 触发相机震动。
  - HP ≤ 0 时设 `NextState = "Dead"`。

### 5.5 `BallisticSystem`

- 职责：根据武器 `BallisticType` 分发 Raycast / Projectile。
- 监听 `IWeaponEvent.OnFire`。
- **Raycast 路径**：
  1. `Physics2D.CircleCast(origin, _tracerRadius, direction, maxRange, Enemy|Obstacle)`。
  2. 命中则构建 `DamageInfo`，触发 `IBattleEvent.OnEntityDamaged`。
  3. 触发 `IHitFeedbackEvent.OnHitTarget(false)`。
  4. 无论是否命中都生成 `TracerVisual`（LineRenderer 曳光弹），按 `tracerSpeed` 飞向命中点。
- **Projectile 路径**：
  1. 若是火箭炮且 `AimAssistSystem.GetLockedTarget()` 不为空，则 `tracking = true` 并记录 `targetId`。
  2. 调用 `ProjectileSystem.CreateProjectile(...)`。
- `Update`：推进所有 active tracer；更新火箭锁定激光（从玩家到锁定目标）。

### 5.6 `ProjectileSystem`

- 职责：飞行物的对象池、飞行、碰撞、命中、回收。
- 关键集合：
  - `_activeProjectiles`：`Dictionary<int, ProjectileData>`
  - `_entityMap`：`Dictionary<int, ProjectileEntity>`
  - `_projectilePool`：内部 `Queue<GameObject>`
  - `_allProjectiles`：所有创建的 GO，用于退出时清理
- 生命周期：
  - `Awake`：创建 `ProjectileRoot`，监听 `OnProjectileHit`。
  - `Start`：异步预加载 20 个 `Projectile_Normal`。
  - `OnDestroy`：销毁所有 `_allProjectiles`。
- 创建：
  1. 从 `MemoryPool` 取出 `ProjectileData`，填充速度、伤害、LayerMask 等。
  2. 从内部池 dequeue 一个 `ProjectileEntity`，若池空且已预加载完成则创建占位体。
  3. `entity.Init(data)`，加入 active 字典，广播 `OnProjectileCreated`。
- 飞行：`Update` 中逐帧 `Tick`：
  - 扣 `LifeTime`。
  - 追踪目标时通过 `Vector2.Lerp` 修正方向。
  - 用 `Physics2D.CircleCast` 做连续碰撞检测，避免穿墙。
  - 命中后入队 `_pendingHits`，统一在帧末处理。
- 命中处理：
  - 普通弹：构建 `DamageInfo` → `OnEntityDamaged`。
  - 爆炸弹（`explosionRadius > 0`）：`OverlapCircleAll` 范围内敌人，按距离衰减伤害。
  - 支持 `PenetrateCount`（穿透计数），当前默认 0。
- 销毁：`DestroyProjectile` → 从字典移除、`entity.OnRecycle()`、回收到 `_projectilePool`、释放 `ProjectileData`、广播 `OnProjectileDestroyed`。

### 5.7 `BattleSystem`

- 职责：伤害结算。
- 监听 `IBattleEvent.OnEntityDamaged`。
- 流程：
  1. 从 `DamageInfo.TargetGameObject` 取 `EnemyEntity`，调用 `TakeDamage(damage, hitDirection)`，并触发命中反馈。
  2. 若目标是玩家自己，调用 `PlayerSystem.TakeDamage`。
  3. `finally` 中 `MemoryPool.Release(damageInfo)`。

### 5.8 `AimAssistSystem`

- 职责：轻微磁吸辅助瞄准 + 火箭锁定。
- 监听 `IWeaponEvent.OnAimStateChanged`。
- 辅助瞄准：
  - 仅在武器 `aimAssistEnabled = true` 且正在瞄准时生效。
  - 搜索 `_aimAssistRadius` 内敌人，取角度最小者，按角度比例 `Lerp` 修正方向。
- 火箭锁定：
  - 仅当持有火箭炮且瞄准时执行。
  - 在 `_lockOnRange`、`_lockOnAngle` 内寻找最佳敌人。
  - 持续瞄准同一目标 `_lockOnHoldTime`（默认 1.5s）后完成锁定，触发 `OnTargetLocked`。
  - `BallisticSystem` 通过 `GetLockedTarget()` 读取锁定目标，并绘制红色激光。

### 5.9 `CameraSystem`

- 职责：平滑跟随、FOV、震动、狙击镜、伤害指示数据。
- 挂载在 `Camera.main` 上。
- 关键功能：
  - `Awake` 创建子相机 `ScopeCamera` 与 512×512 `RenderTexture`。
  - 监听 `OnCameraShake` → 叠加震动幅度/时长。
  - 监听 `OnAimFovChanged` → 平滑 FOV；若目标 FOV ≈ 25（狙击），启用 `ScopeCamera`。
  - 监听 `OnDamageIndicator` → 记录受击角度与持续时间，供 UI 读取。
  - `LateUpdate`：直接读取 `PlayerSystem.Instance.GetPlayerEntity().transform.position` 进行跟随 + 震动偏移；同步狙击镜相机位置/旋转。
  - 跟随模式 `CameraFollowMode`：
    - `Hard`：相机位置与玩家位置完全同步，零延迟，默认模式。
    - `Exponential`：指数平滑，无最大速度上限。
    - `SmoothDamp`：传统 SmoothDamp，有明显平滑滞后感。
  - 注意：相机不再依赖 `OnPlayerPositionChanged` 事件更新目标位置，以避免 `FixedUpdate` 事件频率低于 `LateUpdate` 导致的跟随卡顿。`OnPlayerPositionChanged` 事件仍保留供其他系统使用。

### 5.10 `EnemySpawnSystem`

- 职责：开局生成 3 个敌人占位。
- 配置：`_enemyCount=3`、`_spawnRadius=6`、`_enemyConfigId=9001`、`_enemyMaxHp=50`。
- 流程：
  1. `Start` 异步循环，按 120° 间隔摆放位置。
  2. 加载 `Enemy` 预制体，失败则创建红色占位体。
  3. 确保 `EnemyEntity` 组件，调用 `Initialize(configId, maxHp)`。
  4. 广播 `OnEnemySpawned`。

### 5.11 `HitFeedbackSystem`

- 职责：命中标记 + 8 方向受击指示。
- 监听 `OnDamageIndicator`、`OnHitTarget`。
- 实现：
  - 若 `UIModule.UIRoot` 为空，创建独立 `HitFeedbackCanvas`（ScreenSpaceOverlay）。
  - 否则在 UIRoot 下创建 `HitFeedbackRoot`。
  - 生成 8 个 `Image` 指示器（N、NE、E、SE、S、SW、W、NW），按角度点亮后淡出。
  - 屏幕中心生成 `HitMarker`，命中时闪红/橙 0.15s。

### 5.12 `PoolSystem` / `GameObjectPool`

- 通用 GameObject 对象池。
- `PoolSystem.Get(key, prefab, parent, preloadCount)`：懒创建池并取出一个实例。
- `PoolSystem.Recycle(key, go)`：回收到对应池。
- `PoolSystem.ClearAll()`：清空所有池并销毁对象。
- **注意**：当前 `ProjectileSystem` 使用自己的内部池，未走 `PoolSystem`，后续效果弹/敌人生成可复用此通用池。

## 6. 实体（Entities）

### 6.1 `PlayerEntity`

- 挂载于 `Player` 预制体。
- 组件要求：`Rigidbody2D`；可选 `Animator`/`SpriteRenderer`。
- 关键属性：
  - `MoveDirection` / `AimPosition`
  - `BaseMoveSpeed = 5`、`MoveSpeed`、`DodgeSpeed = 15`、`DodgeDuration = 0.4`
  - `IsMoving`、`IsDead`、`IsDodging`
- 方法：
  - `SetMoveDirection` / `SetAimPosition`
  - `PlayAnimation(stateName)`：映射到 `Player_Idle/Run/Roll/Reload/Dead`。
  - `StartDodge()`：设置 `IsDodging = true`，并设置 RB 速度为 `MoveDirection.normalized * DodgeSpeed`。
  - `EndDodge()`：设置 `IsDodging = false`。
  - `SetDead()`：置 `IsDead`、清零速度、切为 Kinematic。
  - `ResetEntity()`：重置状态。
- `Update`：
  - `transform.up` 转向瞄准方向（渲染帧更新，保证视觉流畅）。
- `FixedUpdate`：
  - 若 `IsDodging` 为 false，应用 `MoveDirection * MoveSpeed` 到 `linearVelocity`；闪避速度由 `StartDodge` 设置，防止被正常移动覆盖。
  - 广播 `OnPlayerPositionChanged`（供非视觉系统使用）。

### 6.2 `EnemyEntity`

- 由 `EnemySpawnSystem` 确保挂载。
- 字段：`ConfigId`、`Hp`、`MaxHp`、`IsDead`。
- 头顶血条：
  - 默认位于敌人头顶上方 `0.6` 单位处。
  - 由 `HealthBarRoot`、`Background`（黑色半透明）、`Fill`（绿/黄/红）三个 `SpriteRenderer` 组成。
  - `EnsureHealthBar()`：优先使用 Prefab 中已配置的血条节点；未配置则运行时动态创建占位血条。
  - `UpdateHealthBar()`：根据 `Hp / MaxHp` 比例缩放填充条，并按血量阈值切换颜色（>60% 绿、>30% 黄、≤30% 红）。
- 方法：
  - `Initialize(configId, maxHp)`：初始化血量并创建/刷新血条。
  - `TakeDamage(int damage, Vector2 hitDirection)`：扣血并刷新血条，0 血时 `Die()`。
  - `Die()`：广播 `OnEnemyDied`，`Destroy(gameObject)`。

### 6.3 `ProjectileEntity`

- 挂载于 `Projectile_Normal` 预制体。
- 字段：`ProjectileData Data`。
- 方法：
  - `Init(data)`：绑定数据并 `UpdateVisual()`。
  - `OnRecycle()`：清空数据、隐藏 Sprite。
  - `UpdateVisual()`：同步位置/旋转、启用 Sprite。
- `OnTriggerEnter2D`：若仍 active，触发 `IProjectileEvent.OnProjectileHit(Data.Id, other.gameObject)`。

## 7. 玩家状态机（FSM）

基类 `PlayerStateBase : FsmState<PlayerEntity>`。

- `OnEnter`：播放动画、广播 `OnPlayerStateChanged`。
- `OnLeave`：记录 `PrevState`。
- `OnUpdate`：读取 FSM 数据 `NextState`，按字符串切换到对应状态（Dodge/Reload/Dead）。

| 状态 | StateName | 行为 |
|------|-----------|------|
| `PlayerIdleState` | Idle | 无输入时停留；`MoveDirection` 非零则切到 Move。 |
| `PlayerMoveState` | Move | 移动中；无输入则切回 Idle。 |
| `PlayerDodgeState` | Dodge | 进入时 `StartDodge()`，离开 `OnLeave` 调用 `EndDodge()`；定时 `DodgeDuration` 后切回 Idle。 || `PlayerReloadState` | Reload | 进入时触发 `IWeaponEvent.OnReload`，定时时间为当前武器 `Config.reloadTime`；完成后切回 Idle。 || `PlayerDeadState` | Dead | 进入时 `SetDead()`，广播 `OnPlayerDied`。 |

- 状态切换保护：`PlayerSystem.CanSwitchFromCurrentState()` 禁止在 Dodge/Reload/Dead 期间被覆盖。

## 9. 流程（Procedure）

热更启动后，流程管理器被重新初始化，注册 `ProcedureMainMenu`、`ProcedureLobby`、`ProcedureBattle`，并从 `ProcedureMainMenu` 开始。

### 9.1 `ProcedureMainMenu`

```
OnEnter
└── EnterMainMenuAsync()
    ├── GameModule.Scene.LoadSceneAsync("MainMenuScene")
    ├── HideLauncherUI()             // 销毁启动加载界面
    └── GameModule.UI.ShowUIAsyncAwait<MainMenuUI>()
```

- `MainMenuUI` 包含“开始游戏”和“退出游戏”按钮。
- 点击“开始游戏”调用 `GameApp.ChangeProcedure<ProcedureLobby>()`。

### 9.2 `ProcedureLobby`

```
OnEnter
└── EnterAsync()
    ├── GameModule.Scene.LoadSceneAsync("LobbyScene")
    └── GameModule.UI.ShowUIAsyncAwait<LobbyUI>()
```

- `LobbyUI` 从 `LevelConfigMgr` 读取所有关卡并生成关卡按钮。
- 点击关卡按钮设置 `BattleContext.CurrentLevelId` 后调用 `GameApp.ChangeProcedure<ProcedureBattle>()`。
- 点击“返回”调用 `GameApp.ChangeProcedure<ProcedureMainMenu>()`。

### 9.3 `ProcedureBattle`

```
OnEnter
└── EnterBattleAsync()
    ├── LevelConfig config = LevelConfigMgr.Instance.Get(BattleContext.CurrentLevelId)
    ├── GameModule.Scene.LoadSceneAsync(config.sceneName)   // 默认 BattleScene，关卡 2 为 BattleScene_L01
    ├── InitializeBattleSystems()    // 创建 BattleRoot + CameraSystem
    └── GameModule.UI.ShowUIAsync<BattleMainUI>()
```

`InitializeBattleSystems` 按以下顺序添加组件：

1. `InputSystem`
2. `PlayerSystem`
3. `WeaponSystem`
4. `AimAssistSystem`
5. `BallisticSystem`
6. `ProjectileSystem`
7. `BattleSystem`
8. `EnemySpawnSystem`
9. `HitFeedbackSystem`
10. `PoolSystem`
11. `CameraSystem`（挂载到 `Camera.main`）

`OnLeave` / `CleanupBattleSystems`：
- 销毁 `BattleRoot`
- `PoolSystem.Instance.ClearAll()`
- `GameModule.Timer.RemoveAllTimer()`
- `GameModule.UI.CloseAll()`
- `GameModule.Resource.UnloadUnusedAssets()`

### 热更入口

`GameApp.StartGameLogic`：
1. 反射调用 `GameModule.Procedure.Shutdown()`。
2. 通过 `ModuleSystem.GetModule<IFsmModule>()` 获取 FSM 模块。
3. 用 `ProcedureMainMenu`、`ProcedureLobby`、`ProcedureBattle` 数组重新 `Initialize` 流程模块。
4. 缓存内部 `_procedureFsm` 字段供 `GameApp.ChangeProcedure<T>()` 使用。
5. `StartProcedure<GameLogic.ProcedureMainMenu>()`。

### 流程切换

- FSM 已启动后，不允许再次调用 `StartProcedure<T>()`（会抛 `FSM is running` 异常）。
- UI 中通过 `GameApp.ChangeProcedure<T>()` 切换流程，其内部通过反射调用 FSM 的 `ChangeState<T>()`。
- 切换前会先执行当前流程的 `OnLeave`，再进入目标流程的 `OnEnter`。

## 12. 扩展与注意事项

### 12.1 新增武器

1. 在 `WeaponConfigMgr` 构造函数中新增配置，分配唯一 `id`。
2. 指定 `weaponType`、`ballisticType`、`fireMode`。
3. 若是 Projectile，填写 `projectileSpeed`、`projectileLifeTime`、`explosionRadius` 等。
4. 在 `WeaponSystem.Start` 或玩家存档初始化中 `EquipWeapon(slot, id)`。
5. 如需新子弹外观，创建新的 prefab 并配置 YooAsset 地址，修改 `ProjectileSystem.TryLoadProjectilePrefab` 的地址逻辑。

### 12.2 新增状态

1. 继承 `PlayerStateBase`，实现 `StateName`。
2. 在 `PlayerSystem.CreatePlayerAsync` 的 FSM 创建参数中传入新状态。
3. 在 `PlayerStateBase.OnUpdate` 的 switch 中新增 `NextState` 映射。
4. 如需保护不可切换，在 `PlayerSystem.CanSwitchFromCurrentState` 中添加判断。

### 12.3 新增事件

1. 在 `IEvent/` 下新增接口方法，标注 `[EventInterface(EEventGroup.GroupLogic)]`。
2. 发布：`GameEvent.Get<IxxxEvent>().OnXxx(...)`。
3. 订阅：`GameEventMgr.AddEvent(IxxxEvent_Event.OnXxx, handler)`。

### 12.4 配置表化

当前 `WeaponConfig` + `WeaponConfigMgr` 为硬编码，后续建议：
- 使用 Luban 生成 `TbWeapon` 等配置类。
- 替换 `WeaponConfigMgr.Instance.Get(id)` 为 `GameModule.Config.GetConfig<TbWeapon>(id)` 或等价 API。
- 保持 `WeaponConfig` 字段命名与 Luban 表一致，可减少 System 层改动。

### 12.5 已知可优化点

- `PoolSystem` 当前未被 `ProjectileSystem` 使用，属于通用池；后续效果弹、掉落物可复用。
- 火箭锁定激光在 `BallisticSystem.UpdateRocketLaser` 中实例化，未走对象池。
- 音效/特效接口已预留字段但未接入，后续接入 `GameModule.Audio` 与特效系统即可。

## 15. 命中反馈与射线 Debug

### 15.1 命中反馈流程

命中反馈统一由 `BattleSystem.OnEntityDamaged` 触发，避免 `BallisticSystem` 与 `BattleSystem` 重复调用：

```
BallisticSystem.FireRaycast / ProjectileSystem.HandleHit
        │
        ▼
IBattleEvent.OnEntityDamaged(damageInfo)
        │
        ▼
BattleSystem.OnEntityDamaged
        ├── 命中敌人：enemy.TakeDamage(...)
        ├── 准心反馈：IHitFeedbackEvent.OnHitTarget(false)
        └── 伤害飘字：DamageNumberUI.Show(damage, screenPos, isCritical)
```

### 15.2 准心命中闪烁

- 由 `HitFeedbackSystem` 实现，在屏幕中心维护一个 `HitMarker` Image。
- `OnHitTarget(bool isCritical)` 时根据是否暴击设置为红色/橙色，并在 0.15s 内淡出。
- 当前 `isCritical` 统一传 `false`，后续接入暴击判定后可区分颜色。

### 15.3 伤害数字飘字

- `DamageNumberUI`：独立的 `UIWindow`，在 `ProcedureBattle` 中随 `BattleMainUI` 一起打开。
- 使用对象池管理飘字 `Text` 实例，避免运行时频繁创建/销毁。
- 调用 `DamageNumberUI.Show(int damage, Vector2 screenPos, bool isCritical)` 后：
  - 将屏幕坐标转换为 `DamageNumberUI` 根节点下的局部坐标。
  - 设置文字、颜色（普通白色 / 暴击橙色）、字号。
  - 在 `OnUpdate` 中执行向上飘移 + 淡出动画，动画结束后回收到对象池。
- 可调整参数（代码中字段）：
  - `_poolSize`：对象池初始大小。
  - `_floatSpeed`：飘移速度。
  - `_fadeDuration`：淡出时长。
  - `_moveOffset`：总飘移距离。

### 15.4 射线检测 Debug 可视化

`BallisticSystem.FireRaycast` 已使用 `Physics2D.CircleCast` 做即时命中检测。为了让策划/程序直观调整，相关参数已下沉到 `WeaponConfig`：

| 字段 | 含义 | 默认值 |
|------|------|--------|
| `raycastRadius` | CircleCast 半径；`< 0` 时使用 `BallisticSystem._tracerRadius` | `-1` |
| `hitLayers` | 检测层；`0` 时使用 `BallisticSystem._hitLayers` | `0` |
| `showDebugRay` | 是否绘制 Debug 射线 | `true`（调试用） |
| `debugRayDuration` | 射线显示持续时间（秒） | `1` |
| `debugHitColor` | 命中时射线颜色 | `Color.green` |
| `debugMissColor` | 未命中时射线颜色 | `Color.red` |

Debug 绘制内容：
- 从枪口到命中点（或最大射程）的主射线。
- 命中点处绘制十字标记。

查看方式：在 Unity Editor 中进入 Play Mode，确保 **Scene 视图 Gizmos 开启** 或 **Game 视图 Gizmos 开启**，即可看到射线。

调整方式：修改 `WeaponConfigMgr` 中对应武器的这些字段，或在将来 Luban 配置表中配置。

## 17. 弹幕扩展与视觉分离方案（提案）

### 17.1 背景

当前 `ProjectileSystem` 中每发子弹对应一个 `GameObject` + `SpriteRenderer`：

```csharp
private readonly Dictionary<int, ProjectileData> _activeProjectiles = new Dictionary<int, ProjectileData>();
private readonly Dictionary<int, ProjectileEntity> _entityMap = new Dictionary<int, ProjectileEntity>();
```

该方案在 M1 阶段足够直观、调试方便，但无法满足后续密集弹幕需求：

- 每个子弹一个 `GameObject`，Draw Call 与 Transform 更新随数量线性增长。
- `Update()` 中每帧 `new List<int>(_activeProjectiles.Keys)` 产生 GC。
- 大量 `CircleCast` 物理查询在敌人密集时开销显著。

### 17.2 目标架构

**逻辑层与视觉层分离**：

- **逻辑层**（`ProjectileSystem`）：只维护 `ProjectileData`，负责飞行、碰撞、命中、生命周期。
- **视觉层**（`ProjectileVisualSystem`）：根据逻辑数据批量渲染：
  - 普通小弹丸 → Particle System
  - 超大规模规则弹幕 → GPU Instancing / Compute Shader
  - 关键子弹（火箭/榴弹/追踪弹）→ 保留实体 GameObject

### 17.3 子弹分类

新增 `ProjectileVisualType`：

```csharp
public enum ProjectileVisualType
{
    Particle,   // 普通小弹丸
    Instanced,  // 超大规模同材质弹丸
    Entity,     // 关键子弹
}
```

| 类型 | 示例 | 视觉方案 |
|---|---|---|
| `Entity` | 火箭、榴弹、追踪弹 | GameObject + ProjectileEntity |
| `Particle` | enemy 散射弹幕、霰弹枪弹丸 | Particle System |
| `Instanced` | 数千发规则弹幕 | GPU Instancing / Compute Shader |

### 17.4 逻辑层改造要点

- `ProjectileSystem` 不再管理 `GameObject` 与 `ProjectileEntity`。
- 用 `List<ProjectileData>` 替代 `Dictionary<int, ProjectileData>`，避免每帧 Keys 分配。
- 命中、销毁事件增加 `ProjectileVisualType` 参数，供视觉层决策。

### 17.5 视觉层实现要点

**Particle System 方案（推荐先落地）**：

- 一个全局 Particle System 管理所有普通子弹。
- 子弹创建时 `Emit(1)`。
- 每帧 `GetParticles()` 读取粒子数组，用逻辑位置覆盖，再 `SetParticles()` 写回。
- Particle System 只负责视觉，命中判定仍由逻辑层 `CircleCast` 完成。

**关键弹**：

- 继续使用对象池 + `ProjectileEntity`。
- `ProjectileVisualSystem` 负责创建、回收、位置同步。

### 17.6 优化策略

**视觉层**：

- 限制 `maxParticles`（如 500），超过上限回收旧子弹。
- 关闭不必要的 Particle System 模块（Color over Lifetime、Collision 等）。
- 单帧大量发射时分帧 `Emit`。
- 超大规模时迁移到 `Graphics.DrawMeshInstanced` 或 Compute Shader。

**逻辑层**：

- 全局子弹数量上限。
- 空间划分（Spatial Hash / Grid）减少碰撞查询。
- 敌人被动检测：子弹只移动，敌人受击盒查询子弹。

### 17.7 实施路线

建议分阶段：

1. **阶段 1**：逻辑层解耦，`Dictionary` 改 `List`，移除 `_entityMap`。
2. **阶段 2**：新增 `ProjectileVisualSystem`，实现关键弹对象池 + 普通弹 Particle System。
3. **阶段 3**：`WeaponConfig` / Luban `TbWeapon` 增加 `projectileVisualType`。
4. **阶段 4**：数量上限、空间划分、Profiling，必要时升级 GPU Instancing。

完整方案见：`docs/Proposal/combat/bullet-logic-visual-separation.md`

# 问题记录 2026-07-17

## 问题现象

Console 刷屏 `Main logic type 'GameApp' missing.`，修复后暴露 `ArgumentException: Key: GameLogic.IBattleEvent`（重复注册）。

## 文档位置

- `.claude/skills/tengine-dev/references/hotfix-workflow.md`：示例中 `GameApp` 为全局命名空间（无 `namespace` 包裹）。
- `.claude/skills/tengine-dev/references/hotfix-workflow.md` / `architecture.md`：未提及 `ProcedureLoadAssembly.OnUpdate` 依赖 `Entrance` 同步完成来终止每帧调用。

## 正确 API（经代码验证）

- 当前工作区（未提交重构）`GameApp` 位于 `namespace GameLogic`，主包反射查找必须使用全限定名：`GetType("GameLogic.GameApp")`（`ProcedureLoadAssembly.cs:135`）。
- `GameEventHelper.Init()` 由 SourceGenerator 生成，**非幂等**：重复调用会在 `EventMgr.RegWrapInterface<T>` 处抛 `ArgumentException`（key 重复）。
- `ProcedureLoadAssembly.OnUpdate` 需一次性守卫（`_assemblyLoadFired`），否则 `Entrance` 每帧被反射调用；异步入口（`await ConfigSystem.LoadAsync()`）会放大该竞态。

## 建议修正

- hotfix-workflow.md 的 GameApp 示例补充说明：若 `GameApp` 被纳入命名空间，主包反射名必须同步改为全限定名（项目当前为 `GameLogic.GameApp`）。
- hotfix-workflow.md "关键约束" 增加一条：`GameEventHelper.Init()` 只允许调用一次；`ProcedureLoadAssembly` 侧须保证 `AllAssemblyLoadComplete` 单次触发。
